### タブの改善点

CursorとGitHub Copilotの最大の違いは、コードを補完する方法にあります。

GitHub Copilotは、カーソル位置にテキストを挿入することができます。カーソル周辺のコードを編集したり、テキストを削除したりすることはできません。

Cursorは、カーソル位置にテキストを挿入できるだけでなく、さらに多くのことができます：
- 複数文字の編集

</Frame>
- 指示ベースの編集
<Frame>
<img src="/images/cpp/instruct.png" />
</Frame>

さらに、Cursorはコンテキストウィンドウに最近の変更履歴を保持しているため、次に何をしようとしているかを把握しています。

### GitHub Copilotからの移行

CursorはデフォルトでGitHub Copilotと一緒に提供されるため、GitHub CopilotとCursorが同時にインストールされている可能性があります。Cursorを使用する際は、GitHub Copilotをオフにすることをお勧めします。

デフォルトでは、CursorがGitHub Copilotよりも優先されます。GitHub Copilotを使用したい場合は、設定で[Cursorを無効化](/tab/overview#copilot-settings) することができます。
